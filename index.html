WCAG 2.1 AA

Accessibility Demo

This wireframe simulates a WCAG 2.1 AA compliant SaaS app. Every pattern — focus management, ARIA roles, keyboard navigation, screen reader announcements — is intentionally implemented to help product and engineering teams understand and validate real-world accessibility requirements.

Keyboard Navigation

Tab / ⇧Tab
Move focus forward and backward
Enter / Space
Activate buttons, links, and controls
↑ ↓ ← →
Navigate menus, tabs, and list items
Esc
Close modals, drawers, and menus
F6 / ⇧F6
Cycle between page landmarks
⌘K
Focus the search bar
?
Toggle the keyboard shortcuts bar

Screen Reader Testing

VoiceOver (macOS)
Cmd+F5 to toggle · VO+U for landmarks rotor
NVDA (Windows)
Ctrl+Alt+N · Insert+F7 for elements list
JAWS (Windows)
H for headings · Semicolon for landmarks
axe DevTools
Browser extension for automated WCAG audits
axe CLI
npx axe [url] --rules wcag2aa
The WCAG 2.1 AA badge in the bottom-right corner lists all implemented criteria and launches the guided accessibility tour.
